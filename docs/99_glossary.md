


# Глоссарий ключевых терминов проекта

## 1. Обзор

Этот глоссарий содержит определения основных терминов и сущностей, используемых в проекте. Он предназначен для быстрой ориентации разработчиков, редакторов и AI-агентов. Используйте его для уточнения значений терминов, встречающихся в документации, коде и интерфейсах системы.

## 2. Основные сущности

- **User** — пользователь системы, обладающий логином, паролем, настройками, ключами Telethon и списком проектов.
- **Project** — рабочая единица, объединяющая источники, параметры фильтрации и публикации.
- **Source** — Telegram-канал или чат, из которого собираются посты.
- **Post** — сообщение, полученное из Telegram, содержащее текст и/или медиафайлы.
- **Story** — сюжет, формируемый из одного или нескольких постов, предназначенный для рерайта.
- **RewriteTask** — задача отправки текста в OpenAI для автоматического переписывания.
- **Publication** — итоговый материал, опубликованный в Telegram-канале.
- **Image** — изображение, сгенерированное для последующей публикации.

## 3. Процессы и термины

- **Parsing** — процесс получения постов из Telegram с помощью Telethon.
- **Deduplication** — проверка и удаление повторяющихся постов.
- **Rewriting** — автоматическое переписывание текста с помощью OpenAI.
- **Prompt** — текстовый запрос, отправляемый модели для генерации ответа.
- **Preset** — предустановленный набор параметров для рерайта.
- **Publication Channel** — Telegram-канал, в который отправляются переписанные посты.
- **Schedule** — отложенная публикация поста на заданное время.
- **FLOOD_WAIT** — ошибка Telegram API при превышении лимитов на отправку сообщений.
- **Retry** — повторная попытка выполнения задачи после возникновения ошибки.
- **Worker** — фоновый процесс, выполняющий задачи (сбор постов, рерайт, публикация и др.).
- **Backoff** — стратегия увеличения задержки перед повторной попыткой после ошибки.

## 4. Статусы и состояния

- **Post Status**:
  - `raw` — пост только что собран из источника.
  - `in_story` — пост включён в сюжет.
  - `processed` — пост обработан и готов к дальнейшим действиям.

- **Story Status**:
  - `draft` — сюжет в черновике.
  - `rewriting` — сюжет в процессе рерайта.
  - `ready` — сюжет готов к публикации.
  - `published` — сюжет опубликован.

- **RewriteTask Status**:
  - `pending` — задача ожидает выполнения.
  - `running` — задача выполняется.
  - `success` — задача успешно завершена.
  - `failed` — задача завершилась с ошибкой.

- **Publication Status**:
  - `scheduled` — публикация запланирована.
  - `sent` — публикация отправлена.
  - `failed` — ошибка при публикации.
