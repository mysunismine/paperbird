# План: экспорт проектов без постов

## Цели
- Дать пользователю возможность выгружать проект в файл без данных постов.
- Сохранить настройки проекта, конфиг промта, источники и снимки пресетов для повторного импорта.

## Объём
- Формат: JSON (при желании YAML как альтернатива).
- Состав экспорта:
  - project: все поля, кроме служебных (created_at/updated_at) и `owner`.
  - prompt_config: секции промта, если есть.
  - sources: поля источников с `web_preset_snapshot`, флагами дедупликации, ретенцией и таймингами.
  - web_presets: опционально активные пресеты, на которые есть ссылки (checksum + config).
- Исключить: posts, stories, worker задачи, Telethon креды пользователя.

## Шаги реализации
1) Продумать формат JSON и правила импорта/конфликтов (что делать при совпадении названия проекта/источника).
2) Добавить вьюху/endpoint `GET /projects/<pk>/export/` с проверкой владельца, возвращать файл `application/json`.
3) Реализовать сериализацию: Project -> dict, PromptConfig -> dict, Sources -> list, WebPreset snapshots -> включать.
4) Покрыть тестами `projects/tests`: доступ только владельцу, состав полей, отсутствие постов, наличие snapshot пресетов.
5) Документация: кратко описать формат и ограничения в README или отдельной заметке.

## Риски и допущения
- Конфликты при будущем импорте: потребуется стратегия (создать новый проект или обновить существующий).
- Размер снапшотов пресетов невелик, поэтому не ожидается проблем с размером файла.
- Без выгрузки постов нет необходимости учитывать медиа-файлы.
