

# Критерии приёмки (Acceptance Criteria) для MVP

## 1. Общие критерии
- Все основные пользовательские сценарии работают end-to-end без ошибок и сбоев.
- После перезапуска приложения данные проектов, постов, сюжетов и настроек сохраняются.
- Все действия пользователя приводят к ожидаемому результату без неожиданных сбоев.

## 2. Аутентификация и пользователь
- **Статус:** критерии покрыты реализованным модулем входа/профиля (октябрь 2025).
- Пользователь может успешно войти в систему с валидными ключами.
- Ключи и токены пользователя надёжно сохраняются и не теряются между сессиями.
- Пользователь может выйти из системы и удалить ключи.
- Настройки пользователя доступны и корректно сохраняются.

## 3. Проекты и сбор постов
- **Статус:** реализовано (модели Project/Source/Post и сборщик `collect_posts`, октябрь 2025).
- Пользователь может создать новый проект и удалить существующий.
- Система не допускает дублирования проектов и постов (дедупликация).
- Реализована фильтрация постов по дате, источнику, ключевым словам.
- Система корректно собирает посты из выбранных источников и сохраняет их в проект.
- Есть возможность очистки медиафайлов из проекта.

## 4. Сюжеты и рерайт
- Пользователь может создать сюжет на основе выбранных постов.
- Для сюжета можно добавить комментарий.
- Результат рерайта возвращается в виде валидного JSON-ответа.
- Доступен предпросмотр текста сюжета до публикации.
- Текст автоматически обрезается при превышении лимита платформы.
- При ошибке рерайта возможен повтор (ретрай) задачи.

## 5. Публикация
- Перед публикацией есть чекбокс для подтверждения.
- Доступен предпросмотр итогового сообщения перед отправкой.
- Публикация происходит отдельными сообщениями для каждого сюжета.
- При ошибке публикации процесс отменяется и отображается уведомление.
- После публикации отображается ссылка на опубликованный пост.

## 6. Генерация изображений
- Генерация изображений происходит в отдельном окне/диалоге.
- Можно добавить комментарий к изображению.
- Доступен превью сгенерированного изображения.
- Возможна перегенерация изображения до нужного результата.
- Сгенерированное изображение можно прикрепить к сюжету.

## 7. Ошибки и уведомления
- Все ошибки отображаются пользователю с понятным описанием.
- Статусы задач (сбор, рерайт, публикация, генерация) видны пользователю.
- О важных событиях и ошибках приходят уведомления.

## 8. Интерфейс (WebUI)
- Реализована навигация между основными разделами (проекты, посты, сюжеты, публикация).
- Есть лента постов с фильтрами по датам, источникам, ключевым словам.
- Поддерживается мультивыбор постов для создания сюжета.
- Предпросмотр сюжета доступен до публикации.

## 9. Нефункциональные критерии
- Время отклика интерфейса не превышает 2 секунд на основные действия.
- Автоматический логаут при неактивности пользователя более 30 минут.
- Интерфейс работает стабильно при одновременной работе с 1000+ постами в проекте.
