

# UI Flows

## 1. Общий обзор
Этот документ описывает пользовательские потоки (user flows) проекта. Он предназначен для быстрой навигации по основным сценариям использования интерфейса и служит ориентиром для разработки, тестирования и обучения пользователей.

## 2. Основной поток работы пользователя
1. Логин пользователя.
2. Переход к списку проектов.
3. Выбор проекта и переход к ленте постов.
4. Создание нового сюжета.
5. Отправка сюжета на рерайт.
6. Просмотр предпросмотра результата.
7. Публикация сюжета.

## 3. Поток работы с проектами
1. Создание нового проекта.
2. Просмотр существующих проектов и их статистики по постам/сюжетам.
3. Переход к ленте постов выбранного проекта.
4. Редактирование выбранного проекта.
5. Удаление проекта.

## 4. Поток работы с лентой постов
1. Применение фильтров к ленте (по дате, статусу и др.).
2. Поиск постов по ключевым словам.
3. Просмотр содержимого отдельного поста.
4. Мультивыбор постов для пакетных действий.
5. Удаление выбранных постов.
6. Переход к созданию сюжета на основе выбранных постов.

## 5. Поток создания сюжета
1. Выбор одного или нескольких постов из ленты.
2. Создание нового сюжета на основе выбранных постов.
3. Добавление комментария или пояснения к сюжету.
4. Отправка сюжета на рерайт.
5. Просмотр предпросмотра рерайт-результата.
6. Редактирование текста при необходимости.
7. Сохранение сюжета.

## 6. Поток публикации
1. Просмотр предпросмотра итогового сюжета.
2. Установка чекбокса подтверждения публикации.
3. Отправка сюжета на публикацию.
4. Отслеживание статуса публикации.
5. Получение ссылки на опубликованный пост в Telegram.

## 7. Поток генерации изображения
1. Открытие страницы изображений сюжета (`/stories/<id>/image/`).
2. Генерация рекомендованного промпта (опционально).
3. Ввод/редактирование описания и запуск генерации.
4. При необходимости — повторная генерация или загрузка/выбор из источников.
5. Прикрепление изображения к сюжету.
6. Выбор изображений для публикации и назначение главного на странице сюжета.

## 8. Поток обработки ошибок
1. Отображение ошибок при возникновении проблем на любом этапе.
2. Возможность повторить задачу или действие.
3. Уведомление пользователя о результатах или необходимости действий.

## 9. Работа с пресетами рерайта
1. Владельцы проекта создают и редактируют пресеты с описанием, стилем и инструкциями.
2. Редактор выбирает нужный пресет в форме рерайта сюжета.
3. При необходимости дополняет пресет собственными комментариями.
4. Запускает рерайт и получает результат, учитывающий выбранные настройки.
